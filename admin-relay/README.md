# لوحة الأدمن + إشعارات الأعطال + الإصلاح الذاتي

هذه المنظومة تربط أعطال المستخدمين (العائلة والأصحاب على قناة الإنتاج) بك أنت
كأدمن، وتتيح إصلاحها تلقائيًا على قناة `/admin/` دون أن تصل الأعطال لأحد.

## كيف تعمل

```
متصفح المستخدم ──(عطل)──▶ Worker (يحمل السر) ──▶ GitHub issue (production-error)
                                     └────────────▶ إشعار Discord/هاتف
                                                          │
                        Routine تلقائي يوقظ Claude ◀───────┘
                        ▼
   يشخّص ← يصلح على فرع source ← ينشر على /admin/ ← يعلّق على الـissue
                        (الإنتاج لا يتغيّر إلا لمّا ترقّي أنت)
```

## الإعداد (مرّة واحدة)

### أبسط خيار — إشعار فقط (بدون Worker)

أنشئ **Discord webhook** (أو بوت Telegram)، وضع رابطه كمتغيّر بناء
`VITE_ABADIOSA_REPORT_URL` في نشر `abadiosa.github.io`. ستصل الأعطال إلى
هاتفك مباشرة. (لا يمكن للإصلاح الذاتي قراءة Discord، فهذا للإشعار فقط.)

### الخيار الكامل — إشعار + إصلاح ذاتي (مع Worker)

1. انشر `worker.js` على Cloudflare Workers (مجاني) واضبط الأسرار المذكورة في
   أعلى الملف: `GITHUB_TOKEN` و`GITHUB_REPO` و`ALLOW_ORIGIN` و(اختياري)
   `DISCORD_WEBHOOK`.
2. ضع رابط الـWorker في `VITE_ABADIOSA_REPORT_URL`.
3. راقِب issues المستودع من تطبيق GitHub على هاتفك لتصلك الإشعارات.

## لوحة الأدمن

مبنية داخل التطبيق على قناة `/admin/` فقط. افتحها بالضغط على شارة **ADMIN**
أسفل يسار الشاشة. تعرض: سجلّ الأعطال على هذا الجهاز، وصحة التطبيق (القناة،
الإصدار، حالة الإشعارات، الشبكة، التخزين)، وزر نسخ التشخيص كاملًا.

## ملاحظة أمان

`VITE_ABADIOSA_REPORT_URL` ينتهي داخل حزمة الويب العلنية. رابط الـWorker/الـ
webhook يُصبح مرئيًا — وهذا مقبول لأنه يستقبل تقارير أعطال فقط ومحدود المعدّل،
ولا يحمل أي سرّ. الأسرار (توكن GitHub) تبقى داخل الـWorker وحده.
